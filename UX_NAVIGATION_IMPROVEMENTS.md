# 🎯 Улучшения навигации и UX в боте

## 📊 Текущее состояние

### ✅ **Правильная реализация (Freepik, Kling)**
- **Webhook-based мониторинг** - результат приходит автоматически через webhook
- **Простые сообщения** - "Генерация запущена, результат придет автоматически"
- **Нет polling** - не нагружает сервер постоянными запросами
- **Чистый UX** - пользователь не видит технических деталей (Task ID, статусы и т.д.)
- **Параллельные генерации** - можно запускать несколько задач одновременно

### ❌ **Проблемная реализация (Midjourney, Runway)**
- **Остатки polling логики** - есть методы `getTaskStatus`, но они не используются
- **Смешанный подход** - webhooks настроены, но UX сообщения отличаются
- **Разные сообщения** - показывают разные детали пользователю
- **Нет единообразия** - каждый сервис выглядит по-своему

## 🔧 Что нужно исправить

### 1. **Унифицировать сообщения о генерации**

#### Текущее состояние Midjourney:
```typescript
const message = `✅ <b>Изображение создается!</b>\n\n` +
  `📝 Промпт: "${text}"\n` +
  `🖼️ Модель: Midjourney ${model}\n` +
  `💰 Потрачено токенов: ${cost}\n\n` +
  `⏳ Генерация займет примерно 1-2 минуты.\n` +
  `🔄 Результат придет автоматически.`;
```

#### Должно быть (как в Freepik):
```typescript
const message = `🎨 <b>Генерация запущена!</b>\n\n` +
  `📝 "${text}"\n` +
  `🖼️ Midjourney ${model}\n` +
  `💰 Стоимость: ${cost} токенов\n\n` +
  `⚡ Можете создавать еще изображения параллельно!\n` +
  `🔔 Результат придет автоматически через 1-2 минуты`;
```

### 2. **Убрать технические детали из сообщений**

❌ **Не показывать пользователю:**
- Task ID
- Статусы (pending, processing, completed)
- Внутренние ошибки API
- Технические логи

✅ **Показывать пользователю:**
- Простое подтверждение запуска
- Ожидаемое время
- Стоимость в токенах
- Понятные ошибки ("Недостаточно токенов", "Неверный промпт")

### 3. **Единообразные кнопки навигации**

Все сервисы должны иметь одинаковые кнопки после генерации:

```typescript
reply_markup: {
  inline_keyboard: [
    [
      { text: '🔄 Еще одно', callback_data: `${service}_quick` },
      { text: '📊 Статистика', callback_data: 'stats' }
    ],
    [{ text: '🏠 Главная', callback_data: 'back_to_main' }]
  ]
}
```

### 4. **Удалить неиспользуемый код polling**

Файлы для очистки:
- `back/src/bot/services/ai/MidjourneyService.ts` - метод `getTaskStatus` (оставить для отладки, но не использовать в production)
- `back/src/bot/services/ai/FreepikLoraService.ts` - метод `pollTaskStatus` (устаревший, заменен на webhooks)

### 5. **Проверить все меню на логичность навигации**

#### Проблемы:
- Некоторые кнопки "Назад" ведут не туда
- Нет единого стиля кнопок
- Разные эмодзи для одних и тех же действий

#### Решение:
- Создать единую функцию `getStandardButtons(context)` для всех меню
- Использовать одинаковые эмодзи для одинаковых действий
- Проверить все цепочки навигации
